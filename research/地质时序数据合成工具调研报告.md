# 地质领域时序数据自动合成工具调研报告

> 调研日期：2026-03-09
> 目标：筛选可直接用于构建地质时序数据合成管道的成熟工具（无需从头训练）

---

## 一、地震波形合成

### 1.1 Instaseis（推荐：大规模批量合成首选）

**开发机构**：ETH Zurich（与 AxiSEM 配套）
**安装**：`pip install instaseis`
**GitHub**：https://github.com/krischer/instaseis
**许可证**：LGPL-3.0

**核心原理**：从 AxiSEM 预计算的格林函数数据库中快速查询合成地震图，响应时间毫秒级（相比 AxiSEM 单次正演需数小时）。

**可生成的数据**：
- 宽频带合成地震图（P波、S波、面波波形时序）
- 地球上任意位置、任意震源机制的波形
- 支持行星应用（火星等）

**输入参数**：
- 预计算的格林函数数据库（由 AxiSEM 生成，可下载现成数据库）
- 震源位置（经纬度、深度）
- 接收器位置
- 震源机制参数（矩张量：走向/倾角/滑动角）

**输出格式**：ObsPy Stream 对象，可导出为 MiniSEED/SAC，形状 `[n_stations, n_timesteps]`
**典型规模**：100~1000 s 时窗，采样率 1~100 Hz

**适合构建**：地震事件波形分类、震相拾取、震源机制反演训练数据集

---

### 1.2 SpecFEM3D（高精度全波形正演）

**开发机构**：Computational Infrastructure for Geodynamics (CIG)
**GitHub**：https://github.com/geodynamics/specfem3d
**语言**：Fortran + C + Python 接口
**许可证**：GPL-3.0

**核心原理**：谱元法（Spectral Element Method），求解弹性波动方程，精度最高。

**可生成的数据**：
- 3D 地质模型中的声波、弹性波、孔隙弹性波波形
- 盆地效应、地形散射、各向异性介质的波形
- 全波形（包含直达波、反射波、面波）

**输入参数**：
- 3D 速度/密度/衰减模型
- 震源矩张量和位置
- 台站网络配置
- 网格参数（六面体网格）

**输出格式**：ASCII/SAC 时间序列，形状 `[n_stations, n_components, n_timesteps]`
**典型规模**：需 HPC，单次正演数分钟到数小时

**适合构建**：区域/局部尺度高精度合成地震数据集（需计算资源）

---

### 1.3 Devito（地震勘探正演，推荐：Python 友好）

**开发机构**：Imperial College London + Chevron
**安装**：`pip install devito`
**GitHub**：https://github.com/devitocodes/devito
**许可证**：LGPL

**核心原理**：基于符号定义的有限差分波动方程自动编译框架，支持 GPU 加速和 PyTorch/TensorFlow 集成。

**可生成的数据**：
- 地震勘探合成炮集（shot gather）时序
- 声波、弹性波、各向异性波正演波场
- 全波形反演（FWI）训练数据
- 逆时偏移（RTM）成像数据

**输入参数**：
- 速度模型（随机层状模型、Marmousi 模型等）
- 震源子波（Ricker 子波，主频 10~50 Hz）
- 接收器排列（偏移距、道间距）
- 网格分辨率和时间步长

**输出格式**：NumPy 数组，形状 `[n_shots, n_receivers, n_timesteps]`
**典型规模**：每炮数百道，时窗 2~8 s，采样率 1~4 ms

**适合构建**：地震勘探反演/成像训练数据集，适合与神经网络联合训练

---

### 1.4 ObsPy（地震数据处理框架，辅助工具）

**安装**：`pip install obspy`
**GitHub**：https://github.com/obspy/obspy
**许可证**：LGPL-3.0

**定位**：不直接生成合成数据，用于后处理 SpecFEM/Instaseis 输出、格式转换、信号处理和数据质量控制。

**主要用途**：
- MiniSEED、SAC、SEG-Y 格式读写
- 滤波、去均值、重采样等预处理
- 合成与真实波形对比分析
- 从 IRIS/FDSN 下载真实地震数据用于对比验证

## 二、储层与地下流体模拟

### 2.1 OpenGeoSys（THMC 耦合，推荐）

**开发机构**：Helmholtz Centre for Environmental Research (UFZ)
**安装**：`pip install ogs`
**GitLab**：https://gitlab.opengeosys.org/ogs/ogs
**许可证**：Modified BSD

**核心原理**：有限元法求解热-水-力-化学（THMC）多物理场耦合过程。

**可生成的数据**：
- 地下水位/压力头时序（水力过程）
- 地层温度场演化时序（热过程）
- 地面沉降/应力应变时序（力学过程）
- 污染物浓度迁移时序（化学过程）
- CO₂ 地质封存注入压力演化
- 核废料储库温度场时序

**输入参数**：
- 水力传导系数、孔隙度、渗透率张量
- 热传导率、比热容
- 弹性模量、泊松比
- 初始/边界条件（温压条件、注入速率）

**输出格式**：VTK、HDF5，可提取为时间序列 CSV
**典型规模**：时间跨度年至百万年，空间分辨率米至千米

**适合构建**：地热储层、CO₂ 封存、地下工程监测时序数据集

---

### 2.2 TOUGH2 / PyTOUGH（多相流体模拟）

**开发机构**：Lawrence Berkeley National Laboratory
**Python 接口**：`pip install pytough`
**应用领域**：地热、油气、CO₂ 封存、水热系统

**可生成的数据**：
- 地热储层温度/压力/饱和度时序
- CO₂ 注入压力演化时序
- 页岩气/水合物开采产量时序
- 多相流体（水-气-油）相态变化时序

**输入参数**：
- 岩石渗透率（1e-18 ~ 1e-12 m²）
- 孔隙度（0.01 ~ 0.35）
- 注入速率、井位坐标
- 初始温压条件（地温梯度 30°C/km）

**输出格式**：ASCII 时间序列，可转 CSV/HDF5
**典型规模**：时间跨度年至千年

**适合构建**：地热/油气储层生产动态时序数据集

---

### 2.3 MODFLOW + flopy（地下水标准工具）

**开发机构**：USGS
**安装**：`pip install flopy`
**GitHub**：https://github.com/modflowpy/flopy
**许可证**：CC0（MODFLOW 本体）

**核心原理**：有限差分法求解地下水流方程，flopy 提供完整 Python 接口。

**可生成的数据**：
- 地下水位变化时序（观测井水位动态）
- 含水层抽水试验响应曲线
- 地下水流量/流速场时序
- 污染物运输浓度时序

**输入参数**：
- 含水层几何（厚度、层数）
- 导水系数、储水系数
- 抽水量/注水量、井位坐标
- 降雨补给量

**输出格式**：二进制 MODFLOW 格式，flopy 读为 NumPy 数组
**典型规模**：时间跨度天至百年，空间网格 10~1000 m

**适合构建**：水文地质监测时序数据集（水位预测、抽水试验分析）

## 三、地球物理正演模拟

### 3.1 SimPEG（多方法地球物理正演与反演）

**开发机构**：UBC-GIF（不列颠哥伦比亚大学）
**安装**：`pip install simpeg`
**GitHub**：https://github.com/simpeg/simpeg
**许可证**：MIT

**可生成的数据**：
- 大地电磁（MT）阻抗/视电阻率时频响应
- 直流电阻率（DC）测深曲线
- 时域电磁（TEM）感应衰减时序
- 瞬变电磁（TDEM）响应时序
- 重力/磁力异常剖面

**输入参数**：
- 地下电阻率/磁化率/密度模型
- 激发源配置（频率范围、发射电流）
- 测量几何（收发距、排列方式）

**输出格式**：NumPy 数组
**适合构建**：地球物理勘探多方法联合数据集（矿产勘探、地下结构探测）

---

### 3.2 pyGIMLi（地球物理反演建模库）

**安装**：`conda install -c gimli -c conda-forge pygimli`
**GitHub**：https://github.com/gimli-org/gimli
**许可证**：Apache 2.0

**可生成的数据**：
- 电阻率法（ERT）视电阻率时序
- 地震折射走时数据
- 感应极化（IP）充电率时序
- 联合多方法正演数据

**特点**：同时支持正演生成和反演验证，适合构建正演-反演配对数据集

## 五、地质建模与测井数据

### 5.1 GemPy（三维隐式地质建模）

**安装**：`pip install gempy[base]`
**GitHub**：https://github.com/gempy-project/gempy
**许可证**：EUPL v1.2

**可生成的数据**：
- 3D 地质体模型（随机参数化批量生成）
- 沿钻孔/剖面的岩性序列（深度序列）
- 地层界面不确定性集合（随机地质模型）

**输入参数**：地层界面点坐标、倾向/倾角、断层参数
**输出格式**：NumPy/xarray
**适合构建**：钻孔岩性序列数据集（地层解释、储层预测）

---

### 5.2 Welly + lasio（测井曲线处理与合成）

**安装**：`pip install welly lasio`
**GitHub**：https://github.com/agilescientific/welly

**可生成的数据**：
- 合成测井曲线（GR、密度、声波、电阻率随深度变化）
- 合成地震道（阻抗 → 反射系数 → 与子波褶积）

**输入参数**：岩石物理参数（孔隙度、泥质含量）、地层厚度
**输出格式**：LAS 文件，可转 pandas DataFrame
**适合构建**：测井解释时序数据集（岩性/流体识别）

## 六、集合数据同化（从少量观测扩充数据）

### 6.1 ERT - Ensemble Based Reservoir Tool

**开发机构**：Equinor
**安装**：`pip install ert`
**GitHub**：https://github.com/equinor/ert
**许可证**：GPL-3.0

**核心原理**：集合平滑器（ES-MDA，Ensemble Smoother with Multiple Data Assimilation），从少量真实观测生成大量物理一致的集合样本。

**可生成的数据**：
- 储层参数（渗透率、孔隙度）集合
- 生产动态（产油量、产水量、井底压力）集合时序
- 不确定性量化下的多种可能轨迹

**适合构建**：小样本真实数据 → 大规模集合数据（数据增强）

## 七、推荐合成路径

根据不同地质场景，推荐以下合成管道：

### 路径 A：地震波形数据集（大规模批量）
```
参数采样（震源机制、震级、深度）
    ↓
Instaseis（毫秒级格林函数查询）
    ↓
ObsPy（滤波、重采样、格式转换）
    ↓
HDF5 存储，形状 [N, n_stations, n_timesteps]
```
**适合任务**：震相拾取、震源机制反演、地震事件分类

---

### 路径 B：地震勘探数据集（与神经网络联合）
```
速度模型采样（随机层状模型、Marmousi 变体）
    ↓
Devito（有限差分波动方程正演，支持 GPU）
    ↓
NumPy 炮集数据，形状 [n_shots, n_receivers, n_t]
    ↓
HDF5 存储
```
**适合任务**：全波形反演（FWI）、地震成像、速度建模

---

### 路径 C：储层/地下水监测数据集
```
岩石物理参数采样（渗透率、孔隙度、边界条件）
    ↓
OpenGeoSys / TOUGH2 / flopy(MODFLOW)
    ↓
压力/温度/水位时序，形状 [N, n_wells, n_timesteps]
    ↓
HDF5/CSV 存储
```
**适合任务**：储层状态预测、地下水位预测、生产优化

---

### 路径 D：地球物理勘探数据集
```
地下电阻率/密度模型采样
    ↓
SimPEG / pyGIMLi（MT、TEM、ERT 正演）
    ↓
视电阻率/感应时序，形状 [N, n_freq_or_t]
    ↓
NumPy/HDF5 存储
```
**适合任务**：地下结构反演、矿产勘探

---

## 八、输入复杂度分析（数据合成管道适用性）

构建自动合成管道时，输入参数的复杂度直接决定了管道设计的难易程度。以下按输入复杂度分三个梯队：

### 第一梯队：输入极简（全部为标量，直接采样即可）

| 工具 | 典型输入参数 | 参数维度 | 适合场景 |
|------|------------|---------|---------|
| **flopy (MODFLOW)** | 导水系数 T、储水系数 S、抽水量 Q、井位 (x,y) | 5~10 个标量 | 地下水位时序 |
| **TOUGH2 / PyTOUGH** | 渗透率 k、孔隙度 φ、注入速率 Q、初始温压 T₀/P₀ | 6~15 个标量 | 储层压力/温度时序 |
| **SimPEG（1D 模式）** | 各层电阻率 ρ（3~10 层）、层厚 h | 6~20 个标量 | 电磁感应衰减时序 |
| **Landlab** | 降雨量 P、侵蚀系数 K、坡度 S、流域面积 A | 4~8 个标量 | 流量/侵蚀时序 |

> **推荐优先使用**：`flopy` + `SimPEG 1D` 是最适合快速构建合成管道的组合，参数物理意义明确，采样分布容易设计。

---

### 第二梯队：输入为小向量/简单 1D 剖面（可接受）

| 工具 | 典型输入参数 | 参数维度 | 适合场景 |
|------|------------|---------|---------|
| **Welly + lasio** | 各地层孔隙度、泥质含量、厚度 | 30~60 个标量（层数×属性数） | 测井曲线时序 |
| **pyGIMLi（1D ERT）** | 各层电阻率和厚度（1D 层状模型） | 10~30 个标量 | 视电阻率测深曲线 |
| **Instaseis** | 震源深度、矩张量（6 分量）、震级、台站坐标 | ~10 个标量 | 地震波形时序 |

> **注意**：Instaseis 输入本身极简，但需要预先下载几十 GB 的格林函数数据库。

---

### 第三梯队：输入为大矩阵（不推荐用于简单合成管道）

| 工具 | 输入复杂度 | 原因 |
|------|-----------|------|
| **SpecFEM3D** | 极高 | 需要完整 3D 速度模型网格（百万节点），还需 HPC |
| **Devito** | 高 | 速度模型是 2D/3D 矩阵，最简单也是 `[nz, nx]` 二维数组 |
| **OpenGeoSys** | 高 | 需要定义有限元网格 + 多物理场参数，配置复杂 |
| **GemPy** | 中高 | 需要手动指定地层界面点云坐标 |

---

## 十、与 PiERN 项目的对应关系

PiERN 的核心是将 LLM 语言推理与物理专家模型无缝结合，上述工具可直接作为**物理隔离专家**的数据来源：

| PiERN 专家类型 | 对应合成工具 | 生成数据用途 |
|--------------|------------|------------|
| FNO 波场预测专家 | Devito / SpecFEM3D | 训练 FNO 专家模型（Stage 1） |
| 储层神经代理 | OpenGeoSys / TOUGH2 | 训练储层代理网络 |
| 地下水预测专家 | flopy (MODFLOW) | 训练水文专家模型 |
| 地球物理反演专家 | SimPEG / pyGIMLi | 训练反演代理网络 |
| Text2Comp 训练数据 | 所有工具 + 语言模板 | 构建（文本描述, 数值张量）配对 |

所有工具均为**纯物理正演，无需预训练**，通过参数采样（类似 PDEBench 策略）即可批量生成高质量带精确数值的时序训练数据。
