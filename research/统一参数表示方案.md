# MODFLOW 统一参数表示方案

**目标**：设计一个精简、统一的参数表示，能够覆盖所有25个场景

---

## 📊 当前问题分析

### 现有参数分布

| 场景类型 | 场景数 | 参数数量 | 主要参数 |
|---------|-------|---------|---------|
| 基础场景 | 15 | 5 | hk, sy, pumping, strt, rch |
| 多层场景 | 2 | 11 | hk_layer1-3, vka, sy, pumping, strt_layer1-3, rch |
| 非均质场景 | 1 | 6 | hk_mean_log, hk_std_log, correlation_length, anisotropy_ratio, sy, pumping |
| 边界场景 | 2 | 8 | 基础5参数 + river_stage/lake_stage, river_cond/lake_cond |
| 季节场景 | 1 | 9 | 基础5参数 + rch_wet, rch_dry, wet_duration, dry_duration |
| 海水入侵 | 1 | 8 | 基础5参数 + coastal_head, freshwater_density, saltwater_density |
| 地面沉降 | 1 | 12 | 多层参数 + elastic_storage, inelastic_storage, precon_head |
| 污染物运移 | 1 | 11 | 基础5参数 + contam_conc, porosity, disp_long, disp_trans, decay_rate, duration |
| 地热储层 | 1 | 13 | 基础5参数 + temp_init, temp_inj, thermal_cond, heat_capacity, ... |

**总计**：25个场景，参数数量从5到13不等

---

## 🎯 统一参数方案设计

### 核心思想

**分层设计**：核心参数 + 可选扩展参数 + 场景标签

```
统一参数向量 = [核心参数(10个)] + [扩展参数(5个)] + [场景元数据(3个)]
总计：18个标量
```

---

## 📋 方案A：最小核心参数集（推荐）

### 1. 核心参数（10个）- 所有场景必需

| 参数 | 符号 | 物理意义 | 范围 | 单位 |
|------|------|---------|------|------|
| **p1** | `K_mean` | 平均水力传导系数 | 0.1-100 | m/day |
| **p2** | `K_std` | 水力传导系数标准差 | 0-50 | m/day |
| **p3** | `K_anisotropy` | 水平/垂向各向异性比 | 1-100 | - |
| **p4** | `S_storage` | 储水系数 | 0.01-0.40 | - |
| **p5** | `Q_pumping` | 抽水量（负值）/注水量（正值） | -1500 to +500 | m³/day |
| **p6** | `H_initial` | 初始水头 | 0.5-48 | m |
| **p7** | `R_recharge` | 平均补给量 | 0.00001-0.010 | m/day |
| **p8** | `R_variation` | 补给量变化幅度 | 0-1 | - |
| **p9** | `BC_strength` | 边界条件强度 | 0-20 | m |
| **p10** | `n_layers` | 含水层层数 | 1-5 | - |

### 2. 扩展参数（5个）- 特殊场景使用

| 参数 | 符号 | 物理意义 | 范围 | 默认值 |
|------|------|---------|------|--------|
| **p11** | `C_concentration` | 污染物浓度/温度 | 0-1000 | 0 |
| **p12** | `D_dispersion` | 弥散系数/热扩散系数 | 0-50 | 0 |
| **p13** | `lambda_decay` | 衰减/冷却系数 | 0-0.1 | 0 |
| **p14** | `phi_porosity` | 孔隙度 | 0.15-0.45 | 0.30 |
| **p15** | `rho_density` | 密度/比重 | 1000-1100 | 1000 |

### 3. 场景元数据（3个）- 分类标签

| 参数 | 符号 | 含义 | 编码 |
|------|------|------|------|
| **m1** | `scenario_type` | 场景类型 | 0=基础, 1=多层, 2=非均质, 3=边界, 4=季节, 5=特殊 |
| **m2** | `output_type` | 输出变量类型 | 0=水头, 1=浓度, 2=温度 |
| **m3** | `complexity` | 复杂度等级 | 1-5 |

### 多层场景（2个）→ 核心参数 + 层数

**映射规则**：
```python

    'K_mean': mean([hk_layer1, hk_layer2, hk_layer3]),  # 平均渗透率
    'K_std': std([hk_layer1, hk_layer2, hk_layer3]),    # 层间差异

    'Q_pumping': pumping,
    'H_initial': mean([strt_layer1, strt_layer2, strt_layer3]),

    'scenario_type': 1,              # 多层
    'output_type': 0,

```

**优势**：
- ✅ 用 `K_mean` 和 `K_std` 统一表示各层差异
- ✅ 用 `K_anisotropy` 统一表示垂向导水性
- ✅ 用 `n_layers` 标签区分单层/多层

## 🎯 方案优势

### 1. 精简性 ✅
- 从最多13个场景特定参数 → 统一18个参数
- 参数复用：`D_dispersion` 用于非均质、边界、污染物、地热
- 参数复用：`lambda_decay` 用于季节周期、衰减、冷却

### 2. 物理一致性 ✅
- `K_mean` 和 `K_std` 统一表示渗透率空间分布
- `R_recharge` 和 `R_variation` 统一表示补给时间变化
- `BC_strength` 统一表示各种边界条件强度

### 3. 可扩展性 ✅
- 核心参数覆盖所有基础物理过程
- 扩展参数处理特殊物理过程
- 元数据标签辅助模型理解场景类型

### 4. 训练友好 ✅
- 固定维度，无需padding
- 参数归一化范围明确
- 场景类型标签辅助条件生成

## 🔧 实施方案

### Step 1: 创建参数转换器

```python
# piern/simulators/modflow/unified_params.py

class UnifiedParamConverter:
    """将场景特定参数转换为统一参数表示"""

    def __init__(self):
        self.param_names = [
            # 核心参数
            'K_mean', 'K_std', 'K_anisotropy', 'S_storage', 'Q_pumping',
            'H_initial', 'R_recharge', 'R_variation', 'BC_strength', 'n_layers',
            # 扩展参数
            'C_concentration', 'D_dispersion', 'lambda_decay', 'phi_porosity', 'rho_density',
            # 元数据
            'scenario_type', 'output_type', 'complexity'
        ]

    def convert(self, scenario_name: str, original_params: dict) -> np.ndarray:
        """转换场景参数到统一表示"""
        if scenario_name in ['baseline', 'low_permeability', ...]:
            return self._convert_basic(original_params)
        elif scenario_name in ['multilayer_3layers', 'multilayer_5layers']:
            return self._convert_multilayer(original_params)
        # ... 其他场景

    def _convert_basic(self, params: dict) -> np.ndarray:
        """基础场景转换"""
        return np.array([
            params['hk'],           # K_mean
            0,                      # K_std (均质)
            1,                      # K_anisotropy
            params['sy'],           # S_storage
            params['pumping'],      # Q_pumping
            params['strt'],         # H_initial
            params['rch'],          # R_recharge
            0,                      # R_variation
            0,                      # BC_strength
            1,                      # n_layers
            0, 0, 0, 0.30, 1000,   # 扩展参数默认值
            0, 0, 1                 # 元数据
        ])
```

### Step 2: 更新数据生成管线

```python
# 在 pipeline.py 中
from piern.simulators.modflow.unified_params import UnifiedParamConverter

converter = UnifiedParamConverter()

# 生成数据时转换参数
for i in range(n_samples):
    original_params = sample_params(cfg)
    unified_params = converter.convert(scenario_name, original_params)

    # 保存统一参数
    unified_params_array[i] = unified_params
```

### Step 3: 更新HDF5存储格式

```python
# 保存数据
with h5py.File(output_path, 'w') as f:
    f.create_dataset('timeseries', data=timeseries)
    f.create_dataset('params', data=unified_params_array)  # 统一参数
    f.create_dataset('param_names', data=converter.param_names)
    f.attrs['n_params'] = 18  # 固定18个
```
